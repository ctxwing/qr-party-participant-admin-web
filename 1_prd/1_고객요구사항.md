## 🎯 프로젝트 목적
파티 현장에서 고객이 QR코드로 접속하여 간단한 상호작용(쪽지, 큐피트, 호감도, 순위 등)을 할 수 있는 모바일웹과, 운영자가 시간/인원/신청상태/닉네임/로그/쪽지/횟수설정을 관리할 수 있는 관리자페이지를 구축하는 것을 목적으로 합니다.

---

## 🛠 업무 내용

### 개발 형태
- **플랫폼**: QR 접속형 모바일웹 (앱스토어 배포 없음)
- **관리자**: 관리자 웹페이지 별도 구축

### 납기 및 진행
- **납기**: 계약일로부터 10일 이내 오픈
- **중간 보고**: 3~5일차 중간 데모 URL 제출 필수
- **최종 기한**: 5월 1일 전 완료 권장 (늦어도 5월 6일까지 최종 완료 필요)

### 필수 사용자 기능
- 파티 진행시간 / 남은시간 표시
- 현재 파티 인원수 표시
- 본인 파티 정보 표시 (예: 1차 신청 O / 2차 신청 X)
- 닉네임 등록 및 변경 (3회 제한)
- **내 현황판**: 큐피트 / 쪽지 / 호감도 / 알림 개수 확인
- 쪽지 보내기
- 큐피트 발사 (기본 2회)
- 호감도 주기 (기본 3회)
- 호감도 순위 표시
- 노래 요청 / SOS 요청
- 파티 나가기

### 필수 관리자 기능
- 파티 시작/종료 시간 설정 (자동 카운트다운 반영)
- 파티 인원수 수정
- 1차/2차 신청 상태 관리
- 닉네임 등록자 중 2차 신청 미완료자 필터링
- 전체 닉네임 조회 및 변경 로그 조회
- 전체 쪽지 조회 (관리자 전용)
- 큐피트/호감도 횟수 조정
- 호감도 순위 화면 관리

### 최종 산출물
- 모바일웹 1식
- 관리자페이지 1식
- 소스코드 전체
- DB 스키마
- 서버 배포본
- 관리자 계정 정보
- 간단 운영 매뉴얼
