# 요구사항 누락 점검 보고서 (최종 업데이트: 2026-04-24)

`1_prd/1_고객요구사항.md` 및 `1_prd/1_고객요구사항_게스트하우스.md`를 바탕으로 한 전수 조사 결과입니다.

## 1. 사용자(참여자) 기능

- [ ] **실시간 파티 진행 시간 및 남은 시간 표시**
    - 현황: DB에는 시간이 저장되나, 사용자 대시보드 및 상단바에 카운트다운 UI가 누락됨.
- [ ] **상호작용 상세 내역 및 알림함**
    - 현황: 수신 개수(Stats)만 표시될 뿐, 누가 보낸 쪽지인지, 어떤 호감도를 받았는지 확인하는 상세 리스트와 과거 알림 확인 기능 누락.
- [ ] **발송 내역 조회**
    - 현황: 본인이 보낸 쪽지, 큐피트, 호감도 내역을 확인하는 페이지 부재.
- [ ] **나의 랭크 강조 표시** (Phase 5 반영)
    - 현황: 랭킹 리스트에서 본인의 위치를 시각적으로(색상, 강조선 등) 구별하는 기능 추가 예정 (T026).
- [x] **파티 나가기**
    - 현황: 대시보드 내 `EXIT` 버튼으로 구현 완료.
- [x] **본인 신청 정보 확인 (1차/2차)**
    - 현황: 대시보드 내 신청 여부 배지 표시 완료.

## 2. 관리자 기능

- [ ] **관리자 상세 제어 페이지 (Party Detail)**
    - 현황: 목록 조회만 가능하며, 특정 파티의 **시작/종료 시간 수정**, **정원(인원수) 수정** 기능 누락.
- [ ] **참석자 상세 관리 및 상태 토글** (Phase 5 반영)
    - 현황: 참여자의 1차/2차 신청 상태를 클릭 한 번으로 변경하는 기능 추가 예정 (T024). 단, **'강제 퇴장' 기능은 원본 요구사항 부재로 제외(미구현사항.md 기록)**.
- [ ] **큐피트/호감도 잔여 횟수 개별 조정**
    - 현황: 특정 참여자에게 횟수를 추가로 부여하거나 차감하는 세부 조정 UI 부재.
- [ ] **실시간 활동 모니터링 피드**
    - 현황: 전체 참여자의 쪽지 발송, 호감도 교환, 닉네임 변경 로그가 실시간으로 흐르는 통합 모니터링 대시보드 부재 (`admin/alerts`는 현재 Mock 상태).
- [ ] **닉네임 변경 이력(로그) 뷰어**
    - 현황: 사용자가 닉네임을 변경한 전후 이력을 추적하여 관리자에게 보여주는 전용 화면 누락.
- [ ] **미완료자 필터링 시스템**
    - 현황: "닉네임 등록자 중 2차 신청 미완료자"를 한눈에 모아보는 필터 기능 부재.
- [ ] **호감도 가중치 관리 UI** (제외)
    - 현황: 원본 요구사항의 '횟수(개수) 조정'으로 대체하며, 가중치 수정 UI는 구현하지 않음.

## 3. 비기능 및 산출물

- [ ] **간단 운영 매뉴얼**
    - 현황: 관리자 및 운영 직원을 위한 시스템 사용 가이드 문서 미작성.
- [ ] **예약 시스템 연동 공간 확보**
    - 현황: 협의 사항이었으나, 향후 확장을 고려한 예약 데이터 연동 필드 및 인터페이스 구조 설계 미흡.

---
**최종 결론**: 핵심 비즈니스 흐름(Happy Path)은 구현되었으나, 실제 운영에 필수적인 **'관리자 제어권(수정/조정)'**과 **'사용자 상세 정보 확인'** 기능이 다수 누락되었습니다. Phase 5를 통해 이를 집중 보완해야 합니다.
