# 파티 관리 및 QR 설정 화면 설계

본 문서는 게스트하우스 파티의 생성, 운영, 이력 관리 및 시스템 설정을 위한 관리자 페이지의 상세 설계를 다룹니다.

## 1. 파티 관리 (Party Management)

### 1.1 화면 개요
- **목적**: 파티의 생명주기(생성 -> 시작 -> 종료)를 관리하고 과거 이력을 추적함.
- **주요 기능**:
    - 단계별 파티 운영 가이드 제공 (상단 플로우 및 팝업)
    - 실시간 파티 생성 및 상태 변경
    - 파티 이력 조회 및 검색 (페이징 지원)
    - 참여자 대상 실시간 공지 발송

### 1.2 상단 영역: 운영 가이드 (Step-by-Step Flow)
상단에는 현재 파티 운영의 흐름을 시각적으로 보여주는 프로세스 바가 위치합니다.

- **프로세스 단계**:
    1. **파티 생성**: 일시 및 기본 정보 설정
    2. **예약 연동**: 기존 홈페이지 예약자 데이터 동기화
    3. **준비 완료**: QR 코드 배치 및 집기 점검
    4. **파티 시작**: 입장 승인 및 실시간 이벤트 진행
    5. **종료 및 정산**: 참여자 피드백 및 데이터 아카이빙

- **가이드 팝업**:
    - [도움말] 버튼 클릭 시 노출
    - 각 단계별 운영자가 체크해야 할 상세 리스트 제공 (예: "2단계: 예약 연동 시 중복 데이터 확인 필수", "4단계: 실시간 공지 기능을 통한 이벤트 안내 권장")

### 1.3 중앙 영역: 검색 및 필터
- **검색 조건**:
    - 파티명 (텍스트 검색)
    - 기간 설정 (시작일/종료일 범위)
    - 상태 필터 (진행 중, 종료, 대기)
- **버튼**: [검색], [초기화], [+ 신규 파티 생성]

### 1.4 하단 영역: 파티 목록 테이블
- **컬럼 구성**:
    - **ID**: 고유 식별자
    - **파티명**: 파티 고유 명칭 (예: "불금 바비큐 파티")
    - **시작 일시**: `YY-MM-DD HH:mm`
    - **종료 일시**: `YY-MM-DD HH:mm`
    - **참여 인원**: 현재 인원 / 최대 인원 (예: `24 / 30`)
    - **상태**: [대기], [진행 중], [종료] (배지 형태)
    - **작업**: [상세보기], [삭제]
- **페이징**: 하단 중앙에 페이지네이션 UI 배치 (10개/20개 단위 선택 가능)

### 1.5 상세보기 (Detail View) 설계
[상세보기] 클릭 시 우측 슬라이드 오버(Slide-over) 또는 전체 화면 전환으로 노출됩니다.

- **기본 정보**: 파티명, 일시, 장소, 설명
- **참여자 현황**: 실시간 참여자 리스트, 닉네임, 입장 시간, 호감도 획득 현황
- **실시간 공지 (Chat Interface)**:
    - 하단에 공지 입력창 배치
    - 발송 시 모든 참여자 앱에 실시간 팝업/채팅 형태로 노출
    - 공지 이력 기록
- **예약 연동 섹션**:
    - [기존 예약 불러오기] 버튼
    - 외부 API 연동 상태 및 데이터 매칭 결과 표시

## 2. QR 관리 (System Settings > QR Management)

### 2.1 위치
- 관리자 페이지 > 좌측 내비게이션 > **시스템 설정** > **QR 관리**

### 2.2 주요 기능 및 UI 구성
1. **앵커 URL 설정**:
    - 현재 배포된 도메인을 기본값으로 자동 로딩 (예: `qr-guesthouse-party.vercel.app`)
    - 관리자가 수동으로 수정 가능한 입력 필드 제공
    - [저장] 버튼

2. **QR 코드 생성기**:
    - **브랜딩**: 중앙에 게스트하우스 로고 삽입 기능 (기본 로고 적용, 필요 시 변경 업로드 가능)
    - **미리보기**: 실시간으로 생성된 QR 코드 확인
    - **하단 텍스트**: QR 하단에 '사이트명'과 '접속 URL' 자동 표시
    - **다운로드**: [PNG 다운로드], [SVG 다운로드], [인쇄하기] 버튼 제공

## 3. 기술적 요구 사항 및 데이터 모델

### 3.1 예약 시스템 연동 (External Integration)
- **방법**: 기존 홈페이지(WordPress 등)의 DB 또는 API에 접근하여 예약자 명단(이름, 전화번호 뒷자리 등)을 동기화.
- **동기화 로직**: 파티 생성 시점에 해당 날짜의 예약자를 Batch 작업 또는 실시간 호출로 가져옴.

### 3.2 실시간 공지 (Real-time Broadcast)
- **기술 스택**: Supabase Realtime 또는 WebSocket
- **메커니즘**: 관리자가 공지 작성 -> `announcements` 테이블 삽입 -> 참여자 클라이언트에서 Realtime 구독을 통해 즉시 노출.

## 4. UI 디자인 컨셉 (Aesthetics)
- **테마**: 프리미엄 다크 모드 (Sleek Dark Mode) 지원
- **컬러**: Indigo (#6366f1)를 포인트 컬러로 사용, 상태별 강조색(Green/Amber/Red) 적용
- **컴포넌트**: Glassmorphism 효과가 적용된 카드 및 모달 UI
- **인터렉션**: 부드러운 트랜지션 및 마이크로 애니메이션 (Hover 효과 등)
