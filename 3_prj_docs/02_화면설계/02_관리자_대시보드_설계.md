# [관리자 대시보드 설계] 고객관리 시스템 전문 설계자의 설계안

**설계 철학**: "관리의 효율성은 데이터의 우선순위에서 나온다"
운영자는 현장의 소란 속에서도 **Critical(긴급)**한 정보를 즉시 파악하고 대응할 수 있어야 합니다. 반응형 설계를 통해 데스크톱과 모바일(스마트폰/태블릿)에서 동일한 제어 능력을 제공합니다.

---

## 1. 주요 화면 및 기능 설계

### [A-01] 실시간 현황 관제 (Real-time Monitoring)
- **UI 구조**:
    - 최상단: [SOS 알림 영역] - SOS 발생 시 화면 전체가 붉게 점멸하며 사운드 발생. (모바일 최우선 노출)
    - 좌측/상단: 주요 지표 요약 (현재 접속자, 1/2차 신청 현황, 누적 쪽지 수)
    - 중앙: 실시간 상호작용 피드 (큐피트, 호감도 발생 내역이 타임라인으로 흐름)
- **UX 포인트**:
    - **즉각 대응**: 알림 클릭 시 해당 참여자의 상세 정보 및 위치(테이블 번호 등) 즉시 확인 가능.

### [A-02] 참여자 및 로그 관리 (CRM)
- **UI 구조**:
    - ag-grid 활용: 강력한 필터링, 정렬, 그룹화 기능 제공.
    - 주요 컬럼: 닉네임, 변경 횟수, 1/2차 상태, 큐피트/호감도 잔여 횟수.
    - 액션 버튼: [횟수 추가], [닉네임 초기화], [강제 퇴장].
- **UX 포인트**:
    - **모바일 뷰 최적화**: 화면 폭이 좁아지면 그리드가 카드 형태의 리스트로 자동 전환되어 터치 조작 용이성 확보.

### [A-03] 세션 및 컨텐츠 제어 (Control Center)
- **UI 구조**:
    - 타이머 설정: 파티 시작/종료 시간 조절 (드래그 또는 퀵 버튼 - "+10분", "즉시 종료")
    - 노래 신청 목록: 참여자들이 보낸 신청곡 리스트 (선택 시 '재생 완료' 처리)
- **UX 포인트**:
    - **실수 방지**: '즉시 종료'와 같은 위험 액션에는 더블 탭 또는 밀어서 잠금해제 방식의 UI 적용.

### [A-04] 반응형 레이아웃 및 모바일 최적화 (Responsive Optimization)
- **데스크톱**: 멀티 컬럼 대시보드로 모든 관제 지표를 한눈에 모니터링.
- **모바일/태블릿**: 
    - **Grid to Card**: 복잡한 표(ag-grid)는 모바일에서 한 줄씩 카드 형태로 스택(Stack)되어 터치 조작이 용이하도록 변환.
    - **Priority Stacking**: SOS 알림 > 현재 세션 시간 > 참여자 관리 순으로 중요도에 따라 상단에서 하단으로 재배치.
    - **Collapsible Menu**: 사이드바 메뉴는 햄버거 메뉴(≡)로 숨겨 작업 영역 확보.
    - **Floating Alert**: 다른 메뉴를 보고 있어도 SOS 발생 시 상단에 플로팅 배너가 나타나 즉각 대응 유도.

## 2. 관리자 시스템 UX 편의성 주의사항 (모바일 최적화 포함)

1. **상태 기반 시각화**: 정상 상태(Green), 주의 상태(Yellow), 긴급 상태(Red)의 컬러 코드를 엄격히 적용하여 인지 부하를 줄입니다.
2. **최소 클릭 원칙**: SOS 해결, 노래 요청 처리 등 반복적인 액션은 단 1번의 클릭/터치로 완료되도록 설계합니다.
3. **모바일 터치 타겟**: 모든 버튼은 최소 44x44px 이상의 크기를 확보하여 현장의 급박한 상황에서도 오동작을 방지합니다.
4. **오프라인 대응**: 네트워크 불안정 시 "연결 끊김" 상태를 명확히 표시하고, 재연결 시 누적된 알림을 순차적으로 요약 보고합니다.
5. **접근 제어**: 관리자 기능은 일반 참여자가 절대 접근할 수 없도록 물리적 경로 분리 및 강력한 세션 만료 정책 적용.

---

## 3. 기술적 고려사항 (Refine 프레임워크 활용)

- **Data Provider**: Supabase Realtime과 연동하여 서버 리프레시 없이 데이터 자동 갱신.
- **UI Library**: shadcn/ui의 고밀도 컴포넌트를 사용하여 관리 효율성 극대화.
- **Responsive**: Tailwind CSS of Breakpoints를 활용하여 모바일 기기에서의 레이아웃 최적화.
