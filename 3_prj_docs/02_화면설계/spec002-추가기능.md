# 화면 설계: 파티 상호작용 제어 및 프리셋 (spec002-추가기능)

본 문서는 파티 운영의 유연성을 위해 추가된 '상호작용 횟수 프리셋' 및 '전체 일괄 지급' 기능에 대한 UI/UX 설계를 정의합니다. 개발 시 본 문서를 최우선 참조하여 구현해야 합니다.

## 1. 파티 설정 - 초기 횟수 프리셋 (Initial Settings)

관리자가 파티를 생성하거나 정보를 수정할 때, 상호작용 기본 횟수를 설정할 수 있는 프리셋 기능을 제공합니다.

### 1.1 UI 구성
- **위치**: 파티 생성/수정 모달 또는 상세 정보 섹션
- **컴포넌트**: Radio Group 또는 Card 스타일의 선택 뷰
- **프리셋 종류**:
    | 프리셋 명 | 호감도 | 큐피트 | 비고 |
    | :--- | :---: | :---: | :--- |
    | **일반 (Default)** | 3회 | 3회 | 일반적인 게스트하우스 파티용 |
    | **열정 (Active)** | 5회 | 5회 | 활동성이 높은 파티용 |
    | **커스텀 (Custom)** | 직접 입력 | 직접 입력 | 관리자가 수동으로 수량 지정 |

### 1.2 UX 정책
- 프리셋 선택 시 하단 '호감도/큐피트 기본 횟수' 입력 필드가 자동으로 채워짐.
- '커스텀' 선택 시에만 입력 필드가 활성화됨.

## 2. 관리자 - 전체 참여자 일괄 지급 (Bulk Grant)

파티 도중 특정 이벤트 타임에 모든 참여자에게 동시에 횟수를 지급할 수 있는 기능입니다.

### 2.1 UI 구성
- **위치**: 참여자 목록(`admin/parties/[id]/participants`) 상단 버튼 영역
- **버튼 명**: `[+] 전체 일괄 지급`
- **팝업(Modal) 구성**:
    - 지급 항목 선택: 호감도 / 큐피트 (체크박스)
    - 지급 수량: 숫자 입력 (기본값 +1)
    - [지급하기] 버튼 (강렬한 액센트 컬러 적용)

### 2.2 UX 정책
- **실시간성**: [지급하기] 클릭 시 Supabase Realtime을 통해 모든 접속 중인 참여자에게 즉시 반영되어야 함.
- **알림**: 지급 완료 시 관리자에게는 성공 Toast, 참여자에게는 "운영자가 호감도를 +1회 선물했습니다!"와 같은 안내 Toast 노출.
