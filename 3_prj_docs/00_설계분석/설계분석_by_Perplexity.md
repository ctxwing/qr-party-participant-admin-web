### 1. 핵심 요구사항 (5가지)

| 항목 | 내용 | 중요도 | 난이도 | 리스크 |
|---|---|---|---|---|
| QR 기반 접속 | QR 스캔 → 모바일웹 진입 | 매우 높음 | 낮음 | 없음 (표준 기술) |
| 실시간 상호작용 | 쪽지, 호감도, 큐피트, 순위 기능 | 매우 높음 | 중~상 | 동시성/실시간 처리 |
| 이벤트 로직 | 횟수 제한, 시간 제한, 상태 관리 | 높음 | 중 | 정책 변경 가능성 |
| 관리자 페이지 | 사용자/로그/설정 관리 | 매우 높음 | 중 | UI 요구 변경 |
| 로그/데이터 추적 | 참여 로그, 메시지 기록 | 중 | 낮음 | 개인정보 처리 |

***

### 2. 필요 기술 스택

빠른 MVP 기준으로 현실적인 조합:

**프론트**
- Next.js 또는 React (모바일웹)
- Tailwind CSS (속도)
- QR 진입 → URL 파라미터 처리

**백엔드**
- Node.js (Express or NestJS)
- 또는 Firebase / Supabase (속도 우선 시 강력 추천)

**실시간 처리**
- Socket.IO 또는 Firebase Realtime DB
- (순위/호감도 업데이트용)

**DB**
- PostgreSQL 또는 Firebase Firestore

**관리자**
- React + 간단한 Admin 템플릿
- 또는 Retool / Supabase Admin (시간 절약용)

**배포**
- Vercel (프론트)
- Railway / Fly.io / Supabase (백엔드)

### 4. 제안 시 차별점

이 프로젝트는 **속도 + 안정성 + 이벤트 경험**이 핵심이라 여기서 차별화하면 먹힘:

**1) 실시간 UX 강조**
- “즉시 반영되는 순위/호감도 UI”
- → 사용자 재미 직결

**2) 운영자 편의성**
- “클릭 몇 번으로 이벤트 설정 변경”
- (시간, 횟수, 기능 on/off)

**3) 장애 대응 설계**
- “현장 이벤트 대비 서버 다운 방지”
  - 예: 서버리스 / autoscale 구조

**4) MVP 전략 제안**
- “1차: 핵심 기능 → 2차 확장”
- 클라이언트 입장에서 신뢰도 상승

**5) 빠른 배포 경험 어필**
- “행사용 웹/이벤트 플랫폼 경험”
- 또는 “3~7일 MVP 납품 사례”
