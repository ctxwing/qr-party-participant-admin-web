# 🏆 실시간 동적 랭킹 시스템 설계 (Ranking Logic)

본 문서는 참여자 간의 상호작용을 기반으로 산출되는 **인기 랭킹 시스템**의 로직과 관리 설정 방법을 설명합니다.

---

## 1. 개요
*   **목적**: 참여자들의 활동(좋아요, 쪽지, 매칭)을 유도하고 파티의 흥미를 높임.
*   **특징**: 관리자가 상황에 따라 랭킹 기준(가중치)을 실시간으로 변경할 수 있는 **동적 산정 방식** 채택.

## 2. 랭킹 산정 공식 (Scoring Logic)

참여자의 최종 점수는 다음과 같은 **가중치 합계(Weighted Sum)** 방식으로 계산됩니다.

$$ \text{Total Score} = (\text{Likes} \times W_{L}) + (\text{Messages Received} \times W_{M}) + (\text{Cupid Matches} \times W_{C}) $$

*   $W_{L}$ : 좋아요 가중치 (기본값: 1)
*   $W_{M}$ : 받은 쪽지 가중치 (기본값: 5)
*   $W_{C}$ : 큐피트 매칭 가중치 (기본값: 10)

## 3. 관리자 설정 (Admin Controls)

관리자는 **Admin Console > 시스템 설정** 탭에서 실시간으로 가중치를 조정할 수 있습니다.

### 설정 항목
| 항목 | 설명 | 영향도 |
| :--- | :--- | :--- |
| **좋아요 (Like)** | 참여자 간의 가벼운 호감 표시 | 낮음 |
| **쪽지 수신 (Message)** | 실제 대화 시도 및 소통 발생 | 중간 |
| **큐피트 매칭 (Cupid)** | 시스템에 의한 매칭 성공 또는 직접 연결 | 높음 |

> **Tip**: 행사 중반에 대화를 장려하고 싶다면 '쪽지 가중치'를 높게 설정하여 쪽지를 많이 받은 사람이 상위권에 노출되도록 유도할 수 있습니다.

## 4. 데이터베이스 구조 (Database)

### `system_settings` 테이블
관리자가 설정한 가중치 값을 영구적으로 보관합니다.
*   **key**: `ranking_weights`
*   **value**: `{"like": 1, "message": 5, "cupid": 10}` (JSON 형식)

### `interactions` 및 `messages` 테이블
점수 계산을 위한 원천 데이터를 제공합니다.
*   **interactions**: `type` 필드(LIKE, CUPID)를 기준으로 가중치 적용.
*   **messages**: 수신자 ID별 개수를 집계하여 가중치 적용.

## 5. 구현 상세 (Implementation)

1.  **실시간 반영**: 관리자가 설정을 저장하는 즉시, 모든 참여자의 랭킹 페이지 점수가 새로운 가중치로 재계산되어 표시됩니다. (Next.js Client-side calculation)
2.  **데이터 동기화**: Supabase Realtime을 통해 새로운 상호작용이나 메시지가 발생할 때마다 랭킹 리스트가 자동으로 갱신됩니다.
3.  **성능 최적화**: 참여자가 접속할 때 최신 가중치와 데이터를 한 번에 가져와 브라우저 내에서 정렬을 수행하여 서버 부하를 최소화합니다.
