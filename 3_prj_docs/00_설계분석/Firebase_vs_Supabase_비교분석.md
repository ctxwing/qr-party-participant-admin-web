# BaaS 솔루션 비교 분석: Firebase vs Supabase

본 문서는 'QR 기반 파티 모바일웹 MVP' 프로젝트의 빠른 개발 적용을 위해 Firebase와 Supabase를 비교 분석하고, 최적의 솔루션을 제안하기 위해 작성되었습니다.

---

## 1. 개요
프로젝트는 10일이라는 짧은 기간 내에 실시간 상호작용(쪽지, 큐피트, 호감도)과 실시간 랭킹 시스템, 그리고 관리자 대시보드를 구축해야 합니다. 이를 위해 서버리스 백엔드 서비스(BaaS) 도입이 필수적입니다.

## 2. 상세 비교 분석

| 비교 항목 | Firebase (Google) | Supabase (Open Source) |
| :--- | :--- | :--- |
| **데이터베이스** | NoSQL (Firestore / Realtime DB) | **Relational (PostgreSQL)** |
| **실시간 기능** | 업계 최고 수준 (기본 제공) | Postgres 변경 사항 구독 (Realtime) |
| **인증 (Auth)** | 익명 로그인 포함 다양한 방식 지원 | **익명 로그인 (Postgres RLS와 연동)** |
| **데이터 쿼리** | 조인(Join) 및 복잡한 집계에 취약 | **SQL 기반 강력한 조인 및 집계 가능** |
| **관리자 도구** | Firebase Console (데이터 중심) | **Refine 등 Admin 프레임워크와 강력한 결합** |
| **확장성/이전** | 구글 생태계 종속성 높음 | 오픈소스 기반, Self-hosting 및 이전 용이 |
| **가격 정책** | 사용량 기반 (예측 어려움) | 티어 기반 (비교적 예측 가능) |

---

## 3. 프로젝트 특성별 검토

### 3.1 실시간 랭킹 및 상호작용 (Ranking & Scoring)
- **Firebase**: NoSQL 특성상 여러 지표(큐피트, 호감도)를 합산하여 실시간 순위를 매기는 로직 구현 시 클라이언트나 Cloud Functions에서 추가 가공이 필요합니다.
- **Supabase**: SQL의 `ORDER BY`, `SUM`, `COUNT` 등 강력한 집계 쿼리를 즉시 사용할 수 있어 랭킹 시스템 구현이 매우 직관적이고 빠릅니다.

### 3.2 관리자 대시보드 (Admin Dashboard)
- **Firebase**: 관리자용 UI를 처음부터 개발해야 하는 공수가 큽니다.
- **Supabase**: **Refine** 프레임워크와 연동 시, DB 스키마만 있으면 관리자 페이지(CRUD, 로그 조회 등)를 수 시간 내에 완성할 수 있어 개발 속도가 압도적입니다.

### 3.3 익명 참여 (QR Anonymous Access)
- 양쪽 모두 익명 인증을 지원하므로 QR 기반 즉시 참여 환경 구현에는 차이가 없습니다.

---

## 4. 최종 추천 및 사유

### **최종 추천: Supabase**

### **추천 사유 (Rationale)**

1.  **실시간 랭킹 구현 효율성**: 본 프로젝트의 핵심인 '호감도 순위'와 '통계' 기능은 관계형 데이터베이스(Postgres)의 집계 기능이 NoSQL보다 훨씬 효율적입니다.
2.  **관리자 페이지 개발 가속화**: 10일의 일정 중 관리자 페이지 개발 기간을 최소화해야 합니다. `Supabase + Refine` 조합은 이 부분에서 Firebase보다 최소 2~3일의 시간을 단축시킬 수 있습니다.
3.  **데이터 무결성**: 닉네임 변경 횟수 제한(3회), 큐피트/호감도 횟수 제한 등 비즈니스 로직을 DB 레벨(Check Constraints, RLS)에서 안전하게 처리할 수 있습니다.
4.  **기술적 부채 방지**: MVP 이후 정식 서비스로 확장 시, 복잡해지는 데이터 관계를 관리하기에는 SQL 기반의 Supabase가 장기적으로 유리합니다.

---

## 5. 결론
초기 설정 속도는 Firebase가 약간 우세할 수 있으나, **'랭킹 시스템'**과 **'관리자 대시보드'**라는 프로젝트의 구체적 요구사항을 고려할 때 **Supabase**가 전체 개발 기간을 단축하고 품질을 높이는 데 더 적합한 솔루션으로 판단됩니다.
