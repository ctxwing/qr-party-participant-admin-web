# 대시보드 세션 초기화 문제 분석 및 해결방안

**작성일**: 2026-04-27  
**문제**: 서버 재기동/화면 리프레시 시 사용자('나')가 식별되지 않아 닉네임, 호감도 등 모든 데이터 초기화

---

## 1. 문제 원인 분석

### 현재 구조
```
┌─────────────────────┐
│  dashboard/page.tsx │
│  (사용자 화면)        │
└──────────┬──────────┘
           │ useStore 사용
           ▼
┌─────────────────────┐
│  useStore (zustand) │
│ - participant: null │  ← 메모리 기반, 새로고침 시 초기화
│ (영속성 없음)       │
└──────────┬──────────┘
           │
           ▼ participant?.id 확인
┌─────────────────────┐
│   fetchData()       │
│ (데이터 조회)        │
└─────────────────────┘
```

### 핵심 문제점

| 계층 | 문제 | 결과 |
|------|------|------|
| **인증** | Better Auth 설정 있음 ✓ | useSession 미사용 ✗ |
| **세션** | useStore는 메모리만 사용 | 새로고침 시 null |
| **데이터** | participant = null | fetchData 실행 안 됨 |
| **UI** | "닉네임 없음, 참여자 0" | 사용 불가 상태 |

**근본 원인**: 사용자 식별 정보가 **메모리에만 저장** → 새로고침 시 손실

## 4. 권장 해결방안: A. Better Auth useSession 활용

### 개요
- 이미 Better Auth 설정 완료
- useSession 훅만 활용하면 됨
- 별도 저장소 불필요
- 서버 세션이 유효한 동안 자동 유지

### 구현 (10분)

#### Step 1: dashboard/page.tsx 상단 수정
```typescript
'use client'

import { useSession } from '@/lib/auth-client'  // ← 추가
import { useStore } from '@/store/useStore'
import { useRealtime } from '@/hooks/useRealtime'

export default function DashboardPage() {
  const { data: session, isPending } = useSession()  // ← Better Auth 세션
  const { participant, setParticipant } = useStore()
  
  // ✓ Step 1: 페이지 로드 시 세션에서 participant 복구
  useEffect(() => {
    if (session?.user?.id && !participant?.id) {
      // Supabase에서 사용자 정보 조회
      const loadParticipant = async () => {
        const { data } = await supabase
          .from('participants')
          .select('*')
          .eq('auth_user_id', session.user.id)  // Better Auth 유저 ID로 조회
          .single()
        
        if (data) {
          setParticipant({
            id: data.id,
            nickname: data.nickname,
            nicknameChangeCount: data.nickname_change_count
          })
        }
      }
      loadParticipant()
    }
  }, [session, participant])
  
  // ✓ Step 2: 로딩 중이거나 세션 없으면 로그인 페이지로
  if (isPending) return <div>로딩 중...</div>
  if (!session) {
    return <Redirect path="/login" />
  }
  
  // 나머지 코드...
}
```

#### Step 2: Supabase 스키마 확인
```sql
-- participants 테이블에 auth_user_id 컬럼 필수
SELECT * FROM participants LIMIT 1;
-- 컬럼 확인: id, nickname, auth_user_id (Better Auth user.id), ...
```

useEffect(() => {
  const stored = localStorage.getItem('participant')
