# 미구현 사항 및 향후 개발 계획 (WordPress 연동 포함)

본 문서는 프로젝트 MVP 단계에서 구현을 보류하거나 향후 확장을 위해 설계된 내용을 기록합니다.

## 1. WordPress (WP) 연동 API 설계 (T017)

### 1.1 연동 목적
- 게스트하우스 통합 관리 시스템(WP 기반)과의 예약 데이터 및 회원 정보 동기화.
- 예약 확정 시 자동으로 파티 참여 QR 코드 발급 및 알림톡 발송 연계.

### 1.2 데이터 흐름 (Data Flow)
1. **WP -> QR Party**: 예약 생성 시 Webhook을 통해 파티 참여 정보 전송.
2. **QR Party -> WP**: 파티 종료 후 참여 성비, 호감도 통계, 랭킹 정보를 WP 대시보드로 전송.

### 1.3 인터페이스 명세 (Draft)
- `POST /api/external/sync-reservation`: WP에서 예약 정보를 전달받는 엔드포인트.
- `GET /api/external/party-stats`: WP에서 파티 통계 데이터를 요청하는 엔드포인트.
- **인증**: API Key 기반 (x-api-key 헤더) 또는 JWT Bearer 토큰.

## 2. 기타 미구현 기능

### 2.1 참여자 사진첩 및 커뮤니티
- 파티 중 촬영한 사진을 업로드하고 공유하는 기능.
- 파티 종료 후에도 유지되는 익명 채팅방.

### 2.2 상세 호감도 매칭 알고리즘
- 단순 클릭 기반이 아닌, MBTI 또는 사전 설문을 활용한 추천 시스템.

### 2.3 다국어 지원 (i18n)
- 외국인 참여자를 위한 영어, 일본어, 중국어 레이아웃 지원.
