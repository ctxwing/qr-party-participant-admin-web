□ 구현 검증 및 산출물
- 저희 팀은 개발 구현 가능성과 성능 검증을 위해 핵심 기능을 프로토타입으로 개발하여 함께 제안 드립니다.
- 실제 작동하는 관리자 대시보드, 참여자 모바일웹, 실시간 공지 발송, 브랜디드 QR 생성 기능을 직접 확인하실 수 있습니다.
- 관리자 화면(10장) 및 참여자 화면(18장)의 스크린샷을 통해 전체 기능 흐름을 확인하실 수 있습니다.

□ 프로젝트 제안 개요
- 귀사의 요구사항(Wishket 공고 #154687)을 면밀히 분석하였습니다.
- 게스트하우스 파티 현장에 최적화된 모바일웹과 관리자 제어 시스템을 통합 제공하는 프로토타입을 준비하여 제안 드립니다.
- 고객 요구사항 대비 100% 구현 완료 상태로, 계약 즉시 상용 배포가 가능한 수준입니다.

□ 제안드리는 3가지 핵심 차별성
① 현장 맞춤형 파티 UX 최적화 설계
- 파티 현장의 어둡고 소란스러운 환경을 고려하여 Glassmorphism 다크 테마, 한 손 조작 기반 UI, Vibration(진동) 피드백을 적용함.
- 익숙하지 않은 참여자도 QR 스캔 직후 별도 안내 없이 즉시 서비스 이용이 가능하며, 관리자는 교육 없이도 파티 현장 제어가 가능함.

② 브랜디드 QR 및 일괄 횟수 지급(Bulk Grant) 관리
- QR 코드를 파티 컨셉에 맞춰 실시간 커스터마이징(색상/해상도 PNG·SVG)하고, 이벤트 타임에 전체 참여자에게 횟수를 일괄 지급하는 기능을 반영함.
- 외부 디자인 툴 없이 현장에서 즉시 QR 인쇄물 제작이 가능하고, 모든 참여자에게 동시 횟수 부여로 파티 흐름이 끊기지 않음.

③ 초저지연 실시간 동기화 및 대용량 데이터 보안 아키텍처
- Supabase Realtime Broadcast 채널로 참여자 상호작용(쪽지, 큐피트, 호감도)과 관리자 공지를 1초 내 동기화함.
- Ag-Grid v35 최적화로 1만 건 이상에서도 0.5초 내 조회 속도를 유지하고, Next.js Middleware 기반 서버 사이드 인증(Better Auth)으로 비인가 접근을 원천 차단함.

□ 현재 프로토타입 기능 요약
- 참여자 모바일웹: QR 입장, 닉네임 설정/변경(3회 제한), 실시간 카운트다운, 인원 수 표시, 1차/2차 파티 신청, 쪽지 발송, 큐피트(2회 제한), 호감도(3회 제한), 실시간 랭킹, SOS/노래 요청, 파티 나가기 구현 완료.
- 관리자 대시보드: 파티 생명주기 관리(생성/시작/종료), 참여자 목록 조회 및 관리, 실시간 공지 발송, 쪽지/호감도/닉네임 변경 이력 모니터링, 횟수 수동 조정, 일괄 횟수 지급, 닉네임 변경 로그, SOS/노래 요청 관리 구현 완료.
- 보안: Supabase Auth 기반 관리자 인증, Next.js Middleware 서버 사이드 경로 보호, RLS(Row Level Security) 적용 완료.
- 기술 스택: Bun 1.1+, Next.js 16.2.4(React 19.2), Supabase(PostgreSQL + Realtime + Auth), Drizzle ORM, Ag-Grid v35, Better Auth, Tailwind CSS v4, shadcn/ui.
