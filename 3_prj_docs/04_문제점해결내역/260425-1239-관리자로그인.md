  # 해결 
  관리자 로그인 시 발생한 TypeError: Failed to fetch 오류와 서버 측의 Failed to initialize database adapter 오류를 분석하고 해결했습니다.

주요 원인 분석
포트 불일치 (브라우저측 오류): .env.local의 BETTER_AUTH_URL이 58100으로 설정되어 있었으나, 현재 서버는 59500 포트에서 실행 중이었습니다. 이로 인해 브라우저가 잘못된 주소로 인증 요청을 보내 Failed to fetch가 발생했습니다.
데이터베이스 연결 문제 (서버측 오류): Supabase의 새로운 IPv4 호환 풀러(pooler.supabase.com) 사용 시, 클라이언트가 SNI(Server Name Indication)를 명확히 전달하지 않으면 "no tenant identifier provided" 오류가 발생할 수 있습니다.
스키마 부재: Better Auth 구동에 필요한 테이블(user, session, account, verification)이 schema.ts 및 실제 DB에 정의되어 있지 않았습니다.
조치 사항
환경 변수 수정: .env.local의 BETTER_AUTH_URL 및 NEXT_PUBLIC_BETTER_AUTH_URL을 현재 서버 포트인 59500으로 업데이트했습니다.
Better Auth 어댑터 개선:
기존의 직접적인 postgres 연결 방식 대신, 프로젝트에서 이미 사용 중인 Drizzle ORM을 활용하도록 @better-auth/drizzle-adapter를 도입했습니다.
src/lib/auth.ts를 수정하여 drizzleAdapter를 사용하도록 변경했습니다.
데이터베이스 연결 보완:
src/lib/db.ts에서 Supabase 풀러 연결 시 SNI 필드가 누락되지 않도록 ssl 옵션에 servername을 명시적으로 추가했습니다.
스키마 정의 추가:
src/lib/db/schema.ts에 Better Auth가 필요로 하는 관리자 계정 및 세션 관련 테이블 정의를 추가했습니다.
다음 단계 (사용자 확인 필요)
현재 개발 서버의 네트워크 환경(사내망)에서 Supabase Postgres 포트(5432)로의 직접 연결이 불안정할 수 있습니다. 만약 서버 실행 후에도 여전히 DB 연결 오류가 발생한다면 다음을 확인해 주세요:

DB 스키마 반영: 추가된 Better Auth 테이블을 실제 DB에 반영해야 합니다.
