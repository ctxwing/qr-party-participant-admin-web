# 문제점 해결 내역: UI 버그 수정 및 대시보드 운영 편의성 고도화

## 1. 발생 정보
- **발생 시각**: 2026년 04월 24일 ~ 25일
- **발생 위치**: 랜딩 페이지, 사용자 대시보드, 랭킹 페이지

## 2. 문제 내용 및 원인 분석
- **문제점 1**: 대시보드 로딩 시 `party_id`가 `undefined`인 경우 Supabase 404 에러 발생.
- **문제점 2**: 랜딩 페이지의 "PARTY" 글자가 네모난 그라데이션 박스로 표시됨 (CSS `background` 단축 속성 오버라이드 이슈).
- **문제점 3**: 대시보드 하단 버튼 중 '노래 신청' 버튼이 누락되었고, 버튼들의 디자인(크기, 색상, 아이콘)이 불일치함.
- **문제점 4**: 하단 버튼의 투명도가 너무 높아 스크롤 시 뒤쪽 내용과 겹쳐 가독성이 떨어짐.
- **문제점 5**: 현재 진행 중인 파티 정보(이름, 시간) 및 참여 인원 수를 확인하기 어려움.

**원인 분석**:
- CSS `background` !important 사용 시 `background-clip` 속성이 초기화됨.
- 컴포넌트 리팩토링 과정에서 일부 UI 요소 및 상태 처리가 누락되거나 방어 코드가 부족했음.

## 3. 해결 체크리스트
- [x] 대시보드 404 에러 (undefined party_id) 방어 코드 추가
- [x] 랜딩 페이지 "PARTY" 텍스트 클리핑 버그 및 배지 텍스트 색상 수정
- [x] 하단 버튼 3종(SOS, MUSIC, RANK) 디자인 통일 및 아이콘 적용
- [x] 버튼 가독성을 위한 `backdrop-filter: blur` 및 불투명도 조정
- [x] 대시보드/랭킹 상단 파티 정보(이름, 시간) 및 총 참여 인원 카운트 표시

## 4. 해결 내역
- **상태**: 해결 완료
- **조치 사항**: 
  1. `dashboard/page.tsx`에서 `party_id` 존재 여부 체크 후 쿼리하도록 수정.
  2. `globals.css`에서 `background` 대신 `background-image`를 사용하여 텍스트 클리핑 버그 해결.
  3. 대시보드 하단 탭을 3분할 균등 레이아웃으로 변경하고 고유 글래스 스타일 적용.
  4. 모든 페이지 상단에 파티 상태(이름, 시작/종료 시간)를 노출하여 운영 가독성 확보.
