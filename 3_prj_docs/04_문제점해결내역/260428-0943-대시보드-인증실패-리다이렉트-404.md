# 일반사용자 대시보드 404 및 무한 리다이렉트 루프

## 발생 시각
- 2026-04-28 09:43

## 문제 현상
1. 일반사용자가 `http://localhost:59500` 접속 시 `/login`으로 리다이렉트 → **404 에러**
2. 1차 수정 후: `/` → `/setup`(0.2초) → `/dashboard`("사용자 정보 복구 중..." 0.2초) → `/` → **무한 리다이렉트 루프**

## 원인 분석

### 인증 체계 분리
본 프로젝트는 **관리자**와 **일반사용자**가 서로 다른 인증 체계를 사용:

| 역할 | 인증 수단 | Hook | 세션 저장 | 보호 경로 |
|------|----------|------|----------|----------|
| 관리자 | Better Auth | `useSession` (`auth-client.ts`) | `better-auth.session_token` 쿠키 | `/admin/*` |
| 일반사용자 | Supabase Auth | `useAuth` (`useAuth.ts`) | Supabase 세션 | `/dashboard` |

### 핵심 원인
`dashboard/page.tsx`가 **일반사용자용 페이지**인데 **관리자용 인증 Hook**(`useSession` from Better Auth)을 사용하고 있었다.

### 무한 루프 발생 흐름
1. `/setup` → Supabase 세션 있음 + 기존 participant 발견 → `/dashboard`로 리다이렉트
2. `/dashboard` → `useSession()`(Better Auth) 호출 → 관리자 세션 없음 → `router.replace('/')` 리다이렉트
3. `/` → 랜딩 페이지 → "파티 참여하기" → `/setup` → Step 1 반복

## 해결 체크리스트

- [x] **문제 1**: 리다이렉트 경로 수정 — `'/login'` → `'/'` (`dashboard/page.tsx`)
- [x] **문제 2**: 인증 Hook 교체 — `useSession`(Better Auth) → `useAuth`(Supabase Auth)
- [x] **문제 3**: 변수명 정리 — `session?.user?.id` → `user?.id` 등 7개소 변경
- [x] **테스트**: `/` 랜딩 페이지 정상 표시 확인
- [x] **테스트**: `/setup` 닉네임 입력 화면 정상 표시 (리다이렉트 없음) 확인
- [x] **테스트**: `/dashboard` 직접 접근 시 정상 렌더링 확인
- [x] **문서**: 문제 해결 내역 업데이트

## 해결 상세 내역

### 수정 1: 리다이렉트 경로 수정
`dashboard/page.tsx` — `router.replace('/login')` → `router.replace('/')`

### 수정 2: 인증 Hook 교체
`dashboard/page.tsx` 전체 7개소 변경:

| 항목 | 변경 전 | 변경 후 |
|------|--------|--------|
| import | `useSession` from `@/lib/auth-client` | `useAuth` from `@/hooks/useAuth` |
| 호출 | `{ data: session, isPending: sessionLoading }` | `{ user, loading: sessionLoading }` |
| Step 1 세션체크 | `session?.user?.id` | `user?.id` |
| Step 1 쿼리 | `session.user.id` | `user.id` |
| Step 1 의존성 | `[session, ...]` | `[user, ...]` |
| Step 2 인증없음 | `!session` | `!user` |
| fetchData 가드 | `session?.user?.id` | `user?.id` |
| fetchData 의존성 | `[participant, session, ...]` | `[participant, user, ...]` |

## 결과
- `/` 접속 시 랜딩 페이지 정상 표시 (404 없음)
- `/setup` → 닉네임 입력 화면 정상 표시 (리다이렉트 없음)
- `/dashboard` 직접 접근 시 Supabase 인증 체크 후 대시보드 정상 표시
- 무한 리다이렉트 루프 해결

## 교훈
- 일반사용자 페이지(`/dashboard`)와 관리자 페이지(`/admin/*`)의 인증 체계가 다르므로, Hook import 시 반드시 용도를 확인할 것
- `useSession` = Better Auth (관리자), `useAuth` = Supabase Auth (일반사용자)
