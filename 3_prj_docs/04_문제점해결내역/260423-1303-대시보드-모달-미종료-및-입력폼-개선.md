# 문제 해결 내역: 대시보드 모달 미종료 및 입력 폼 기능 개선

## 1. 문제 발생 개요
- **발생 시각**: 2026-04-23 12:14
- **문제 현황**: 
    - 노래 신청 및 SOS 요청 후 모달 창이 자동으로 닫히지 않음 (입력값만 초기화됨).
    - 입력창이 고정 높이여서 긴 내용 입력 시 불편함.
    - 글자 수 제한 및 현재 입력 상태 확인 불가.

## 2. 원인 분석
- **모달 미종료**: 
    - 상태 변경 함수(`setIsSosOpen` 등)가 비동기 로직 및 리액트 배칭(Batching) 과정에서 확실하게 렌더링에 반영되지 않거나, 폼 컴포넌트 내부에서 제어권이 모호했음.
    - `DialogTrigger`와 커스텀 버튼 간의 이벤트 충돌 가능성.
- **입력 폼 제한**:
    - 기존 `Input` 컴포넌트는 단일 행 전용으로 설계됨.
    - 실시간 글자 수 계산 로직 부재.

## 3. 해결 방안 (체크리스트)
- [x] 자동 높이 조절이 가능한 `GrowingTextarea` 컴포넌트 구현
- [x] 각 폼별 글자 수 제한(MAX_LENGTH) 적용 및 카운터 UI 추가
- [x] 폼 컴포넌트에 `onClose` 콜백을 명시적으로 전달하여 종료 보장
- [x] 상태 업데이트 시 `setTimeout` 등을 활용해 확실한 렌더링 사이클 확보 (보강)

## 4. 해결 내역 (2026-04-23 13:19 업데이트)
- **Textarea 고도화**: `prj_source/frontend/src/components/ui/textarea.tsx`를 수정하여 `onInput` 이벤트 발생 시 `scrollHeight`를 기준으로 높이가 자동 조절되도록 구현. 기본 1행 시작.
- **모달 제어 로직 수정**: 
    - `SosForm`, `GlobalMusicForm` 등의 컴포넌트에 `onClose` prop을 추가하여 버튼 클릭 시 부모의 상태를 확실히 변경하도록 수정.
    - `handleSOS`, `handleSongRequest` 함수 내에서 `setTimeout`을 통해 상태 변경을 비동기적으로 처리하여 모달이 닫히지 않는 현상 해결.
- **UI 개선**: 텍스트 영역 우측 하단에 `현재글자수/최대글자수` 표시 추가 및 최대치 도달 시 경고 색상 적용.

## 5. 향후 방안
- 이후 유사한 `Dialog` 기반 작업 시, 제어 컴포넌트(Controlled Component) 방식에서 `onClose` 콜백을 명시적으로 사용하는 패턴을 유지할 것.
- 멀티라인 입력이 필요한 모든 곳에 `Textarea` 컴포넌트 활용 권장.
