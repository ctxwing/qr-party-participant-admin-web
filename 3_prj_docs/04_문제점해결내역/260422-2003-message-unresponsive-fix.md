# 문제점 해결 내역: 쪽지 전송 무반응 및 Base UI 경고 보완

**작성 일시**: 2026-04-22 20:03
**상태**: **해결 완료**

## 1. 발생한 문제점

### 이슈 1: 대시보드 쪽지 보내기 무반응
- **내용**: 사용자가 쪽지 발송 버튼을 눌렀을 때 응답이 없거나 팝업이 닫히지 않아 전송 여부를 알 수 없음.
- **원인**: 전송 시마다 세션 ID를 조회하는 비동기 로직의 지연 및 다이얼로그 제어 상태 누락.

### 이슈 2: Base UI DialogTrigger nativeButton 반전 경고
- **내용**: 실제 `<button>` 요소를 사용하는 트리거에 `nativeButton={false}`를 적용하여 발생하는 역방향 경고.

## 2. 원인 분석 및 해결 방안

| 문제점 | 상세 원인 | 해결 방안 |
| :--- | :--- | :--- |
| **이슈 1** | `await supabase.from(...).single()`의 잦은 호출 및 비제어 다이얼로그 | 세션 ID를 페이지 로드 시 캐싱하고, `openInteractionId` 상태로 다이얼로그 제어 |
| **이슈 2** | 자식 요소 타입과 `nativeButton` 설정 불일치 | `Card`는 `false`, `Button` 컴포넌트는 `true`(기본값)로 설정 최적화 |

## 3. 해결 체크리스트
- [x] `sessionId` 상태 추가 및 페이지 로드 시 초기화 로직 구현
- [x] `handleSendMessage` 및 `handleInteraction` 내 세션 ID 캐시 사용 및 다이얼로그 닫기 로직 추가
- [x] 참여자별 다이얼로그 제어를 위한 `openInteractionId` 상태 도입
- [x] SOS, 노래 신청, 닉네임 변경 트리거에서 `nativeButton={false}` 제거 (기본값 버튼 사용)
- [x] 브라우저 콘솔 로그 최종 확인 (모든 경고 제거됨)

## 4. 해결 내역 요약
1. **성능 최적화**: 쪽지 및 상호작용 전송 속도를 개선하기 위해 세션 조회를 최소화하였습니다.
2. **UX 개선**: 전송 성공 시 팝업을 자동으로 닫아 사용자에게 명확한 피드백을 제공합니다.
3. **표준 준수**: Base UI의 요구사항에 맞춰 시맨틱 태그와 속성을 일치시켜 접근성 경고를 해결했습니다.
