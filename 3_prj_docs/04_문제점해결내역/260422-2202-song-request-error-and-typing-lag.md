# 문제점 해결 내역: 노래 신청 400 에러 및 타이핑 지연 현상

**발생 일시**: 2026년 04월 22일 21:46
**작성 일시**: 2026년 04월 22일 22:02

## 1. 발생한 문제점
1. **노래 신청 시 400 Bad Request 에러 발생**
   - 대시보드 하단의 'Music' 버튼 또는 특정 유저 대상 노래 신청 시, Supabase `/rest/v1/alerts` 엔드포인트로 POST 요청이 실패함.
2. **노래 제목 입력 시 타이핑 지연 (버벅임) 현상 발생**
   - 곡 제목을 입력할 때마다 화면 렌더링이 심하게 지연되어 입력이 매끄럽지 않음.

## 2. 원인 분석
1. **400 Bad Request 원인 (DB 스키마 불일치)**
   - `alerts` 테이블의 `receiver_id` 컬럼이 `UUID` 타입으로 선언되어 있으나, 프론트엔드 코드 및 `participants` 테이블의 실제 `id`는 `TEXT` 타입임.
   - 코드에서 `alerts` 테이블에 `session_id`를 함께 insert 하려고 시도하나, 실제 테이블에는 해당 컬럼이 누락되어 있음.
2. **타이핑 지연 원인 (리액트 렌더링 최적화 누락)**
   - `songTitle`, `sosMessage`, `message` 등의 입력 상태(State)가 대시보드 최상단 컴포넌트(`DashboardPage`)에 선언되어 있음.
   - 키보드를 한 번 누를 때마다 수십 명의 참여자 목록과 다이얼로그가 포함된 거대한 최상단 컴포넌트가 전체 재렌더링되면서 병목이 발생함.
