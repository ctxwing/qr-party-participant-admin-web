# Better Auth 어댑터 초기화 실패 (Failed to initialize database adapter)

## 발생 시각
- 2026-04-26 10:24

## 문제 현상
- `src/lib/auth.ts`를 Kysely 어댑터에서 Better Auth 내장 어댑터(`type: "postgres"`)로 변경 후 브라우저 접속 대기는 풀렸으나,
- 서버 콘솔에 `[Error [BetterAuthError]: Failed to initialize database adapter]` 에러 출력.
- 브라우저에서 로그인 요청 시 500 에러를 반환.

## 원인 분석
- `src/lib/auth.ts`에서 `database: { type: "postgres", url: process.env.DATABASE_URL! }` 설정을 사용할 경우, Better Auth는 내부적으로 `pg` 패키지를 데이터베이스 드라이버로 사용함.
- `package.json`을 확인한 결과 `pg` 모듈이 설치되어 있지 않고 `postgres` 모듈만 설치되어 있어 드라이버 로드에 실패함.
- **연관 오류:** 과거(260425-1239) 문서에 따르면 Turbopack에서 `pg` 모듈 사용 시 번들링 오류가 발생한다는 기록이 있음. (`pg`는 Turbopack에서 번들링 불가)

## 체크 리스트
- [x] 원인 분석 완료 (`pg` 모듈 누락)
- [ ] 해결 방안 결정 및 `pg` 모듈 설치 (옵션 A)
- [ ] `next.config` 설정으로 Turbopack 호환성 문제 해결
- [ ] 브라우저 무한 대기 현상 해결 여부 테스트

## 해결 방안 검토

### 옵션 A: `pg` 패키지 설치 및 Turbopack 우회 (채택 ⭐️)
- **개요:** `bun add pg`를 통해 공식 패키지를 설치하고, Next.js의 `serverExternalPackages` 설정을 통해 Turbopack 번들링 오류를 회피합니다.
- **장점:** Better Auth의 안정적인 내장 드라이버(`type: "postgres"`)를 그대로 쓸 수 있어 향후 유지보수와 프레임워크 호환성이 가장 뛰어납니다. 무한 대기 버그도 구조적으로 해결됩니다.

### 옵션 B: `postgres.js` 기반 커스텀 어댑터 구조 개선
- **개요:** 기존 방식(Kysely + `postgres.js`)으로 롤백한 뒤, `pg-pool-adapter.ts`에서 발생하는 Connection Pool 무한 대기(트랜잭션 락 등) 코드를 디버깅하고 고치는 방식입니다.
- **단점:** Pooler 환경에서 발생할 수 있는 엣지 케이스들을 직접 제어해야 하므로 불안정할 수 있습니다.

### 결과 및 추가 조치 (최종 해결)
- 옵션 A( `pg` 패키지 설치 )를 시도하였으나, `next.config.ts`에 `serverExternalPackages`를 추가했음에도 불구하고 Turbopack 개발 서버에서 다음과 같은 번들링 오류가 발생했습니다.
  `Error: Failed to load external module pg... ResolveMessage: Cannot find package 'pg'`
- 결국 **Next.js Turbopack 환경에서는 `pg` 패키지를 완전히 배제**해야만 하는 것으로 결론이 났습니다.

**최종 우회 방안:**
기존에 문제가 되었던 커스텀 `pg-pool-adapter`의 트랜잭션 충돌 문제를 피하기 위해, `postgres.js`를 완벽하게 지원하는 공식 Kysely 다이얼렉트인 **`kysely-postgres-js` 패키지를 설치**하여 적용했습니다.

```typescript
// auth.ts 최종 변경 내역
import { PostgresJSDialect } from "kysely-postgres-js";
import { client } from "./db";

export const auth = betterAuth({
  database: new PostgresJSDialect({
    postgres: client,
  }),
```
이 방식을 통해 Turbopack 번들링 오류(`pg` 누락)와 무한 대기 현상(Custom Pool 어댑터 결함)을 모두 피할 수 있게 되었습니다.
