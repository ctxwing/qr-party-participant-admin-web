# 문제 해결 내역: 관리자 페이지 UI 고도화 및 데이터 동기화 안정화

**최종 업데이트 시각**: 2026-04-23 17:20 (1차 기록 16:11)

## 1. 발생한 문제점

### UI/UX 및 디자인 관련
- **메뉴 바 오버플로 및 정렬 오류**: 메뉴 테두리와 활성 메뉴 배경(Pill)이 어긋나거나, 모바일 환경에서 메뉴가 잘리는 반응형 미지원 문제.
- **슬라이드 스위치 시각적 직관성 부족**: '해결' 여부에 따른 색상 구분이 모호하고, 현재 상태(완료/대기)를 텍스트로 즉시 확인할 수 없음.
- **현황판 정보 밀도 부족**: 요약 카드가 3개로 제한되어 있고, SOS나 노래신청의 상세 처리 현황(완료/미완료)을 확인하기 어려움.
- **날짜 형식 불일치**: 각 테이블마다 날짜 표시 형식이 달라 관리자 가독성 저하.

### 데이터 및 백엔드 통신 관련
- **Supabase 스키마 캐시 오류 (PGRST200/204)**: 실제 DB 구조와 API 캐시 불일치로 인해 조인(`join`) 쿼리 및 컬럼 업데이트(`is_resolved`) 실패.
- **실시간 데이터 닉네임 누락**: SOS 수신 시 요청자 정보가 "시스템"으로만 고정되어 표시되는 현상.
- **이벤트 간섭**: AgGrid의 행 선택 이벤트가 내부 스위치 클릭 이벤트를 방해하여 조작 불능 상태 발생.

## 2. 원인 분석

- **디자인 구조 결함**: 고정 패딩과 높이 설정으로 인해 자식 요소가 부모 테두리를 이탈하거나 반응형 대응 실패.
- **컬럼명 오기**: 코드에서 사용한 `is_resolved`가 실제 DB에는 `resolved`로 정의되어 있어 업데이트 시 `PGRST204` 에러 발생.
- **PostgREST 동기화 지연**: Supabase의 스키마 캐시가 최신화되지 않아 서버 측 조인이 원활하지 않음.
- **Realtime 데이터 한계**: 실시간 페이로드에는 관계 데이터(Join 대상)가 포함되지 않아 클라이언트 측에서 매핑 로직 누락 시 정보 손실.

## 3. 해결 체크리스트 및 결과

- [x] **메뉴 바 반응형 및 정렬 최적화**: `flex-wrap` 적용, 패딩 비율 조정, 글래스모피즘(`backdrop-blur`) 강화로 정렬 및 디자인 문제 해결.
- [x] **스위치 UI 고도화 및 색상 반전**: 해결 시 **파스텔 녹색**, 미해결 시 **강조 보라색**으로 변경하고 우측에 **'완료/대기'** 라벨 및 애니메이션 추가.
- [x] **4종 요약 카드 및 실데이터 연동**: 참여자/메시지/SOS/노래신청 카드로 재구성하고, 각각 완료/미완료 건수를 실시간 데이터로 집계하여 표기.
- [x] **수동 조인(Manual Join) 도입**: 스키마 캐시 에러를 근본적으로 회피하기 위해 클라이언트 사이드에서 참여자 정보를 직접 매핑하는 방식으로 로직 전면 개편.
- [x] **DB 컬럼명 정정**: `is_resolved` 참조를 모두 실제 DB 명칭인 `resolved`로 수정하여 업데이트 기능 복구.
- [x] **실시간 동기화 보완**: `INSERT`뿐만 아니라 `UPDATE` 이벤트도 수신하여 다른 관리자의 처리 현황이 즉시 반영되도록 개선.
- [x] **날짜 형식 통일**: 전역적으로 `YY-MM-DD HH:MM:SS` 형식을 사용하도록 `formatDate` 유틸리티 적용.

## 4. 향후 주의 사항

- **Supabase 스택 관리**: DB 스키마 변경 시 반드시 Supabase 대시보드에서 **Reload Schema**를 실행하거나 DDL 명령으로 캐시 갱신을 유도할 것.
- **AgGrid 상호작용**: Grid 내부의 인터랙티브 요소는 반드시 `e.stopPropagation()`을 사용하여 부모 이벤트와의 간섭을 원천 차단할 것.
- **데이터 무결성**: 실시간 수신 데이터가 불완전할 경우, 페이로드의 ID를 기반으로 즉시 단일 조회를 수행하여 완전한 데이터 객체를 구성할 것.
