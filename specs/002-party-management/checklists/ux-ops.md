# UX 및 운영 관리 Checklist: 파티 관리 및 QR 설정

**Purpose**: 파티 운영 프로세스의 UX 완성도와 실시간 관리 기능의 요구사항 품질을 검증합니다.

**Feature**: [spec.md](../spec.md)

**Note**: 이 체크리스트는 `/speckit.checklist` 명령을 통해 파티 관리 기능의 요구사항 품질을 점검하기 위해 생성되었습니다.

## 1. 요구사항 완전성 (Completeness)

- [x] CHK001 파티 생성 시 필수 입력 항목(명칭, 시작/종료 시간, 최대 인원)에 대한 제약 조건이 정의되어 있는가? [Data Model]
- [x] CHK002 예약 시스템 연동 시 데이터 불일치(중복, 누락)를 해결하기 위한 관리자 개입 프로세스가 포함되어 있는가? [Spec §Edge Cases]
- [x] CHK003 파티 이력 삭제 시 연결된 데이터(참여자 활동, 공지 내역)의 처리 방침(완전 삭제 vs 아카이빙)이 명시되었는가? [Spec §US2: 삭제 전 확인 절차 포함]

## 2. 요구사항 명확성 (Clarity)

- [x] CHK004 '실시간 공지' 발송 시 참여자에게 노출되는 UI 형태(팝업, 푸시, 채팅창 상단 고정 등)가 구체적으로 기술되었는가? [Spec §US1: 즉시 공지 팝업 노출]
- [x] CHK005 상단 가이드 플로우의 각 단계별 '준비 사항'에 들어갈 구체적인 텍스트 내용이나 템플릿이 정의되어 있는가? [Resolved: 구현 시 기획서 문구 기반 작성]
- [x] CHK006 QR 코드 하단에 표시될 '사이트명'의 글자 수 제한이나 폰트 크기 가이드가 있는가? [Spec §FR-006: 규격 정의 완료]

## 3. 요구사항 일관성 (Consistency)

- [x] CHK007 관리자 대시보드의 기존 색상 체계(Indigo, Dark Mode) 및 컴포넌트 라이브러리(shadcn/ui)와 일관성을 유지하는가? [C-005 준수]
- [ ] CHK008 파티 상태 배지(대기, 진행 중, 종료)의 컬러 코드가 시스템의 다른 상태 표시와 충돌하지 않는가? [Gap: 컬러 정의 필요]

## 4. 시나리오 및 엣지 케이스 (Coverage)

- [x] CHK009 파티 종료 시간이 지났음에도 관리자가 종료 처리를 하지 않을 경우의 자동 종료 로직이 정의되었는가? [Resolved: 수동 종료로 확정]
- [x] CHK010 네트워크 장애로 인해 실시간 공지 발송이 실패했을 때의 재시도 UI 또는 알림이 설계에 포함되었는가? [Spec §Edge Cases]
- [x] CHK011 QR 코드의 앵커 URL이 수정되었을 때, 이미 인쇄되어 배포된 구버전 QR 코드에 대한 리다이렉션 처리가 고려되었는가? [Resolved: 리다이렉션 로직 적용]

## 5. 측정 가능성 (Measurability)

- [ ] CHK012 '1초 이내 도달'이라는 실시간성 기준을 검증하기 위한 구체적인 측정 지표나 로그 기록 방식이 있는가? [Gap: 측정 도구 미정]
