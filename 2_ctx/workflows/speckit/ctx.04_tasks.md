# 프로젝트 Tasks 관리 지침 (ctx.tasks.md)

## 🚨 에이전트 필수 준수 및 (T) 생성 규칙

에이전트는 `/speckit.tasks.md` 명령을 수행하여 실제 작업용 `tasks.md` (T)를 생성할 때, 아래 규칙을 **모든 세부 Task에 강제로 포함**해야 한다.

1. **테스트 패스 의무 (Definition of Done)**: 단순히 코드를 작성하거나 항목을 추가하는 것은 완료가 아니다. **정의된 테스트(단위/Smoke/E2E)를 실행하고 성공(Pass)한 결과가 확인되어야만** 해당 Task를 완료(`[x]`) 처리할 수 있다.
2. **모든 Task 하위 필수 항목 (필수, 기계적 삽입)**:

- `[ ] 관련 테스트(단위/Smoke/E2E) 실행 및 통과 확인 (결과 로그 확인 필수)`
- `[ ] 3_prj_docs/완료내역/TASK_<X>_완료내역.md 작성 (테스트 성공 결과 포함)`
- `[ ] 현재 tasks.md 내 해당 항목 완료 체크([x])`
- `[ ] 한글 메시지로 git commit 및 git push 수행`

3. **제약 사항**: 모든 문서는 **한글**로 작성하며, STEP 2 완료 전까지 UI 구현은 엄격히 금지한다.

4. **충돌 및 문제 보고**: Task 수행 중 설계/지시 간 충돌이 발견되거나 기술적 문제가 발생하면 **즉시 사용자에게 보고하고 확인을 받은 후 진행한다**. 임의로 판단하여 진행하지 않는다.

5. **프로젝트 폴더 구조 지시**: 산출물의 폴더는 ./prj_source 아래 ./prj_source/frontend, ./prj_source/backend, ./prj_source/database 등의 형식으로 진행하도록 한다. 이 폴더 외 루트 레벨 폴더가 필요한 경우 반드시 사용자에게 확인 후 진행한다.

6. tasks.md 를 생성후 다음단계로 `/speclit.clarify` 또는 `/speclit.analyze` 진행하도록 권고한다.
