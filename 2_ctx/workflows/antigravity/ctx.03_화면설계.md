# 화면 설계 지침

- 요구사항 기반 웹 화면 구성 (분석 표 + Text Mockup 결합)을 작성한다.
- 반드시 PC웹/ 모바일 웹 이 둘다 가능하도록 반응형으로 개발한다. 
- 핵심 주요화면만을 우선 설계한다.

- 아래의 내용은 다른 프로젝트의 예시임으로 본 프로젝트의 내용과는 다르다. 다만 형식 참고용이다. 

# 참고예시:  화면 설계 내용 
### 📱 1. 메인 화면 (실시간 응급실 혼잡 지도)

| 영역 | 구성 요소 (기능) | 상세 내용 (텍스트 Mockup) |
| :--- | :--- | :--- |
| **A. 상단 헤더** | 로고 및 검색/필터 | `[응급실 ZERO] [🔍] 환자 유형 검색 (Button)` |
| **B. 실시간 지도 영역** | 지도 및 위치 핀 | **(지도 영역)** 서울/경기 지역 중심. 핀 색상으로 혼잡도 표현 (🟢, 🟡, 🔴). 사용자 현재 위치 📌 표시. |
| **C. 하단 혼잡도 범례** | 범례 정보 | `🟢 낮음 | 🟡 보통 | 🔴 높음` |
| **D. 현재 위치 버튼** | 재탐색 기능 | `[🔄] 내 주변 응급실 다시 찾기 (Button)` |
| **E. 응급실 리스트** | 미리보기 카드 (Swipe Up) | (지도 하단 스와이프 영역)<br> **가까운 순 3개 미리보기:**<br> `- [병원명A], 🔴 높음, 3.5km`<br> `- [병원명B], 🟡 보통, 5.1km` |

-----

### 📱 2. 환자 유형 검색 화면 (전문 병원 탐색)

| 영역 | 구성 요소 (기능) | 상세 내용 (텍스트 Mockup) |
| :--- | :--- | :--- |
| **A. 상단 헤더** | 제목 및 닫기 | `[X] 환자 유형 검색` |
| **B. 검색 입력창** | 텍스트 입력 필드 | `[환자 유형, 증상, 전문 분야 검색] (Input Field)` |
| **C. 유형별 버튼** | 빠른 검색 태그 | `[소아] [외상] [심혈관] [뇌졸중] (Button Group)` |
| **D. 검색 결과 영역** | 리스트 | **(결과 목록)**<br> `- [병원명C] (4.2km), 소아 병상 2개, 🟡 보통`<br> `- [병원명D] (6.0km), 외상 전문, 🔴 높음` |

-----

### 📱 3. 응급실 상세 정보 화면

| 영역 | 구성 요소 (기능) | 상세 내용 (텍스트 Mockup) |
| :--- | :--- | :--- |
| **A. 상단 헤더** | 뒤로가기 및 제목 | `[<] [병원명 (선택된 병원)]` |
| **B. 병원 기본 정보** | 상태 표시 | **병원 주소:** [주소 정보]<br> **현재 혼잡도:** **🔴 매우 붐빔 (강조)** |
| **C. 정량적 혼잡도 정보** | 주요 지표 테이블 | **응급실 대기 환자 수:** [X]명<br> **현재 진료 가능 병상 (일반):** [Y]개<br> **응급실 포화도 지수:** [Z]% |
| **D. 환자 유형별 가용 병상** | 전문 병상 정보 | **소아 전용 병상:** [A]개<br> **수술실 가용 여부:** [가능]<br> **CT 촬영 가능 여부:** [가능] |
| **E. 추가 행동 버튼** | 전화 및 길 안내 | `[📞 전화 걸기] (Button) [🚗 길 안내 시작] (Button)` |
