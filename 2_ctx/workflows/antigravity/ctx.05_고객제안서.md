# ctx.고객제안서작성.md

아래의 지시에 따라 고객에게 제안하는 제안서를 txt형식으로 작성한다.

## Steps

 - 아래 지정된 "예시: 참고제안서" 내용을 형식만 참고하여 3_prj_docs/1_고객제안서_제출용.txt 를 작성한다.
 - 문서의 형식은 txt이고 bullet은 예시와 같이 사용한다. 
 - 최대한 간결하고, 명확하고, 설득력있게 작성한다.
 - 핵심 차별성을 반드시 3 가지 도출한다. 
 - 차별성의 내용은 제공하는 기능, 고객이 체감할 수 있는  효용과 그 근거를 2 문장이내로 작성한다.
 - 반드시 동영상 링크 내용을 포함한다. 
 - 개발기간 및 내용은 반드시 이 프로젝트의 1_prd/ 디렉토리 내 설계분석 문서의 내용을 참고하여 작성한다

 # 예시: 참고 제안서
``` 
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
의료 검사 데이터 통계 분석 보고서 자동화 플랫폼 구축 제안
(프로토타입 기반 최종 제안서)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
- 업계 경험 보유: 일양약품 신약 OOO임상 설문결과 시스템 구축 및 2년간 운용유지 중에 있습니다.

□ 개발 검증 및 산출물
- 저희는 개발 구현 가능성과 설계 검증을 위해 완전한 프로토타입을 개발하여 함께 제안 드립니다.
- 첨부된 동영상1, 동영상2 데모를 꼭 참조하시어 실제 작동하는 시스템의 모든 핵심 기능을 직접 확인하여 주시기 바랍니다.
(이메일, 전화번호, 회사명 등 정보 없는 데모 영상입니다)
https://drive.google.com/drive/folders/1gbzLVaVvic-6n4Et_4Cz3xCqni9azRHg?usp=sharing

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

□ 프로젝트 제안 개요
- 귀사의 요구사항(Wishket / Freemoa 공고 및 첨부 자료)를 면밀히 분석하였습니다.
- 존 Excel 매크로의 한계를 완전히 극복하는 고도화된 통계 분석 솔루션 프로토타입을 완성하여 제안 드립니다.

□ 제안드리는 핵심 3가지 차별성
① 의료 데이터 특화, WLS 다중 옵션 기반 통계 정확성 극대화
- 의료 검사 데이터의 농도별 편차를 고려하여 WLS 옵션 4종을 통합 지원합니다.
- python 통계분석 모듈로 통계적 정확성을 확보하며, 신뢰성 있는 모델을 자동으로 선택합니다.
- 자동 PASS/FAIL 판정 기능(허용한계, 퍼센트, ADL 기준)을 완벽하게 구현했습니다.

② 사용자 편의성을 고려한 시스템, 샘플데이터 제공
- 한 번 등록으로 다수의 분석 물질 통계 데이터를 효율적으로 관리하는 고도화된 구조입니다.
- Excel 파일 다중 행 업로드를 지원하여 운영 효율성을 대폭 향상시킵니다.
- 병원/검사실/장비별 익명화 벤치마킹 데이터 관리가 용이하며, 표준 양식 샘플 다운로드 및 검증 로직을 제공합니다.

③ Executive 대시보드 기반 실시간 분석 현황 관리
- 분석 결과 통계(PASS/FAIL), Peer Data 현황, 분석률 등을 실시간으로 조회하는 통합 대시보드입니다.
- 관리자의 신속한 의사결정을 위한 정량적 Insight 제공 및 시스템 상태 모니터링이 용이합니다.
- 분석 완료/진행 중/오류 시각화 및 Peer Group별 추세 그래프 반영이  가능합니다.
- 권한별 접근제어 (PeerData는 관리자만 접근 하도록 제어가능)

□ 개발 검증 및 산출물
- 저희는 개발 구현 가능성과 설계 검증을 위해 완전한 프로토타입을 개발하여 함께 제안드립니다.
- 첨부된 동영상 데모를 꼭 참조하시어 실제 작동하는 시스템의 모든 핵심 기능을 직접 확인하여 주시기 바랍니다.

□ 현재 프로토 타입 기능 요약
- 핵심 분석 엔진: OLS 및 WLS(4종) 예시 및 자동 PASS/FAIL 판정 로직 완성.
- 보고서 자동 생성: Excel 생성 ( 차트/표 자동 삽입 기능 및 PDF 변환)
- 데이터 관리: Peer Data 메타 데이터 , 본데이터 분리 저장, 검색, 필터링 기능.
- 사용자 인터페이스: React 기반 UI/UX, 다크 모드 지원, 실시간 대시보드.
- 1) 서버~PC웹 구조 기 구현 : React.js(UI), Python(Statsmodels/Pandas 분석 엔진), PostgresQL(DB).
- 2) 데스크탑앱 기 테스트 실시: Tauri(Desktop), React.js(UI), Python(Statsmodels/Pandas 분석 엔진), SQLite(DB).크로스 플랫폼 배포(Windows/macOS/Linux)가 용이


□ 총 개발 기간은 6주(약 40일) 소요될 것으로 예상합니다.

1단계: 상세 기획 및 설계 (1주)
2단계: Peer Data 관리 기능 고도화 (1주)
3단계: 핵심 분석 로직 최적화 및 안정화 (2주)
4단계: 통합 테스트 및 품질 검증 (1주)
5단계: 운영 가이드 및 최종 배포 (1주)
```
