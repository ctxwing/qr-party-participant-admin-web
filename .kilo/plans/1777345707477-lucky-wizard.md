# 일반참여자(Participant) 모바일 스크린샷 캡처 계획

## 개요
- **대상**: 일반참여자(Participant) 모바일 웹 화면 (세로 모바일 뷰포트)
- **닉네임**: `천상여구` → `천상여우` (닉네임 변경 테스트 포함)
- **타겟 참여자**: `바람남`
- **브라우저 뷰포트**: 모바일 세로 (390 x 844, iPhone 14 Pro 기준)
- **저장 경로**: `3_prj_docs/screenshots/participants/`
- **서버**: `http://localhost:59500`
- **캡처 방식**: Playwright MCP 브라우저 → `browser_take_screenshot` (viewport only, NOT fullPage)

## 전제조건
- 서버가 기동되어 있어야 함 (`http://localhost:59500`)
- Supabase에 활성 파티 세션 존재
- `바람남` 참여자가 이미 등록되어 있어야 함 (없으면 setup에서 생성)
- DB에 테스트 데이터(참여자, 메시지 등)가 충분해야 함

## 실행 순서

### Step 0: 사전 준비
1. 브라우저 뷰포트를 모바일 세로로 설정 (390 x 844)
2. `http://localhost:59500` 접속 확인
3. Supabase에 `바람남` 참여자 존재 여부 확인 (없으면 별도 브라우저에서 admin으로 생성 또는 API로 생성)

### Step 1: 랜딩 & 온보딩 캡처 (Phase 1)
1. `/` 접속 → `01-landing-page.png` 캡처
2. "파티 참여하기" 클릭 → `/setup` 이동 → `02-setup-nickname.png` 캡처
3. 닉네임 입력란에 `천상여구` 입력 → `03-setup-nickname-filled.png` 캡처
4. "파티 참여하기" 제출 → `/dashboard` 이동

### Step 2: 대시보드 캡처 (Phase 2)
1. 대시보드 로딩 완료 대기 → `04-dashboard-main.png` 캡처
2. 스크롤하여 스탯 카드 영역 → `05-dashboard-stats-cards.png` 캡처
3. 참여자 목록 → `06-participant-list.png` 캡처

### Step 3: 상호작용 테스트 (Phase 3)
1. `바람남` 참여자 카드 클릭 → 상호작용 다이얼로그 → `07-interaction-dialog.png` 캡처
2. "호감 보내기" 클릭 → `08-send-heart-to-baram.png` 캡처
3. 다시 `바람남` 클릭 → "큐피트 보내기" 클릭 → `09-send-cupid-to-baram.png` 캡처
4. 다시 `바람남` 클릭 → 쪽지 작성 → `10-send-message-to-baram.png` 캡처 후 전송

### Step 4: 쪽지 보관함 (Phase 4)
1. Inbox 카드 클릭 → 쪽지 보관함 다이얼로그 → `11-message-inbox.png` 캡처

### Step 5: 닉네임 변경 (Phase 5)
1. 프로필 영역(닉네임) 클릭 → 닉네임 수정 다이얼로그 → `12-nickname-edit-dialog.png` 캡처
2. `천상여우`로 변경 → 변경 완료 후 → `13-nickname-changed.png` 캡처

### Step 6: SOS & 노래신청 (Phase 6)
1. 하단 SOS 버튼 클릭 → SOS 다이얼로그 → `14-sos-dialog.png` 캡처
2. 하단 MUSIC 버튼 클릭 → 노래 신청 다이얼로그 → `15-music-request-dialog.png` 캡처

### Step 7: 랭킹 (Phase 7)
1. 하단 RANK 버튼 클릭 또는 `/ranking` 이동 → `16-ranking-page.png` 캡처
2. 뒤로 가기 → `/dashboard` 복귀

### Step 8: 사이드바 이력 (Phase 8)
1. 우상단 삼선(☰) 메뉴 클릭 → 사이드바 오픈 → `17-sidebar-menu.png` 캡처
2. "수신 호감" 메뉴 클릭 → `18-sidebar-received-hearts.png` 캡처

### Step 9: README.md 반영
1. `3_prj_docs/screenshots/participants/` 에 18장 이미지 저장 확인
2. README.md에 `## 📸 스크린샷` 섹션 추가
3. 이미지 18장 → 3개 컬럼 레이아웃으로 테이블 구성
4. 각 이미지에 한글 설명 캡션 추가

## 주의사항
- **viewport only 캡처**: `fullPage: false` (16:9 가이드이나 모바일 세로이므로 9:19.5 비율)
- **모바일 에뮬레이션**: `browser_resize`로 390x844 설정
- 각 액션 후 데이터 반영(Realtime) 대기 시간 확보 (1-2초)
- 토스트 메시지가 캡처에 포함되도록 액션 직후 캡처
