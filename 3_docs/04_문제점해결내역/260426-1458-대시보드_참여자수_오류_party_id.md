# 대시보드 Active Participants 참여자 수 오류 - party_id 컬럼 미존재

## 문제 발생 시각
- 2026-04-26 14:58 (KST)

## 문제 증상
- 관리자1, 사용자1, 사용자2 총 3명 접속 중
- 각 사용자 대시보드의 Active Participants가 **1명**으로 표시
- 정상이라면 본인(1) + 나머지(1) = **2명**이어야 함

## 원인 분석

| 항목 | 내용 |
|------|------|
| 근본 원인 | `participants` 테이블에 `party_id` 컬럼이 존재하지 않음 |
| 발생 배경 | 대시보드 코드에서 `.eq('party_id', me?.party_id \|\| '')` 필터 사용 → `me.party_id`가 항상 `undefined` → 빈 문자열로 필터 → 결과 0건 |
| 결과 | `(count \|\| 0) + 1` = `0 + 1` = 1명만 표시 |

### 문제 코드 (`dashboard/page.tsx`)
```ts
// party_id 컬럼이 participants에 없어 항상 빈 결과 반환
const { data: others, count } = await supabase.from('participants')
  .select('id, nickname, last_active', { count: 'exact' })
  .eq('party_id', me?.party_id || '')  // ← 컬럼 미존재
  .neq('id', participant.id)
```

## 해결 체크리스트

- [x] 1. `.eq('party_id', me?.party_id || '')` 필터 조건 제거
  ```ts
  // 수정 후
  const { data: others, count } = await supabase.from('participants')
    .select('id, nickname, last_active', { count: 'exact' })
    .neq('id', participant.id)  // 본인 제외만 필터
    .order('last_active', { ascending: false })
    .limit(40);
  ```
- [x] 2. 참여자 수 정상 표시 확인 (본인 제외 +1 = 정상 인원)

## 참고 사항
- 기존 `participants` 테이블 스키마에 `party_id` 컬럼 없음 (id, nickname, created_at, last_active, session_id, anonymous_id, nickname_change_count, is_first_applied, is_second_applied, cupid_count, like_count)
- 향후 파티별 참여자 분리가 필요한 경우 `party_id` 컬럼 추가 및 관련 로직 재검토 필요
- DB 스키마와 코드에서 참조하는 컬럼이 일치하는지 초기 설정 시 반드시 확인 필요
