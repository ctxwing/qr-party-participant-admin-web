# 관리자 페이지 컴포넌트 분리 및 기존 타입 에러 일괄 해결

## 발생 시각
- 2026-04-26 16:00 ~ 16:18 (해결)

## 문제 현상
1. `admin/page.tsx`가 950줄 단일 파일로 5개 탭(현황판/공지/참여자/쪽지/설정), 15개 useState, 10개 fetch 함수가 몰려 있어 유지보수 불가
2. `admin/parties/[id]/page.tsx:158` - `DialogTrigger`에 base-ui 미지원 `asChild` prop 사용으로 타입 에러
3. `dashboard/page.tsx:221` - `updateNickname` 반환값에 존재하지 않는 `historyWritten` 프로퍼티 접근으로 타입 에러
4. `lib/pg-pool-adapter.ts` - Better Auth 어댑터 전환 후 미사용 파일 잔존, `PoolClient.query` 타입 불일치 에러
5. `tests/` 디렉토리 - `bun:test` 모듈 미인식으로 `tsc --noEmit` 시 12건 에러 (tsconfig에서 exclude 누락)

## 원인 분석
- **문제 1**: 초기 MVP 개발 시 빠른 구현을 위해 단일 파일에 모든 로직을 배치
- **문제 2**: Radix UI 기반(`asChild`)에서 base-ui 기반(`render` prop)으로 Dialog 컴포넌트 교체 후 기존 사용처 업데이트 누락
- **문제 3**: `updateNickname` 서버 액션이 이미 내부에서 `nickname_history` INSERT를 수행하도록 개선되었으나, 대시보드 측에서 중복 저장 로직을 제거하지 않음
- **문제 4**: `kysely-postgres-js`로 전환 완료 후 `pg-pool-adapter.ts` 삭제 누락
- **문제 5**: 테스트 파일을 `tsconfig.json` exclude에 포함하지 않아 `bun:test` 타입 선언 미발견

## 해결 내역

### ✅ 1. 관리자 페이지 컴포넌트 분리
기존 `admin/page.tsx` (950줄) → 8개 파일로 분리

| 파일 | 줄수 | 역할 |
|------|------|------|
| `app/admin/page.tsx` | 130 | 인증 + 레이아웃 쉘 + 탭 라우팅 |
| `hooks/useAdminData.ts` | 115 | 공유 데이터 fetch, 실시간 구독, 액션 핸들러 |
| `lib/admin-utils.ts` | 13 | `formatDate` 공통 유틸 |
| `components/admin/DashboardTab.tsx` | 120 | 현황판 탭 |
| `components/admin/AnnouncementsTab.tsx` | 115 | 실시간 공지 탭 (자체 상태) |
| `components/admin/ParticipantsTab.tsx` | 230 | 참여자 관리 + 상세 모달 |
| `components/admin/MessagesTab.tsx` | 70 | 쪽지 모니터링 탭 |
| `components/admin/SettingsTab.tsx` | 95 | 시스템 설정 탭 (자체 상태) |

**공통화**: 참여자/알림/메시지 데이터 fetch, 실시간 구독, 세션 관리 → `useAdminData` 훅 일원화

### ✅ 2. DialogTrigger asChild → render prop 변경
- 파일: `app/admin/parties/[id]/page.tsx:158`
- 변경: `<DialogTrigger asChild>` → `<DialogTrigger render={<Button ... />}>`

### ✅ 3. dashboard 닉네임 변경 중복 이력 저장 제거
- 파일: `app/dashboard/page.tsx:221`
- 서버 액션 `updateNickname`이 이미 내부에서 이력을 INSERT하므로 클라이언트 측 중복 코드 제거

### ✅ 4. 미사용 pg-pool-adapter.ts 삭제
- 파일: `lib/pg-pool-adapter.ts` 삭제
- 이미 `kysely-postgres-js`로 전환 완료되어 참조 없음

### ✅ 5. tsconfig.json에 tests 디렉토리 exclude 추가
- 파일: `tsconfig.json`
- 변경: `"exclude": ["node_modules"]` → `"exclude": ["node_modules", "tests"]`
